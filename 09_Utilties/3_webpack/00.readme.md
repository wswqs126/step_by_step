# 为什么要用 webpack

## 前端越来越复杂

随着前端业务逻辑的复杂，文件拆分不可避免，这样便于开发和维护。但是，为了上线，又有打包的需求，webpack 可以用来打包。

## 关于打包

打包，其中可做的事情实际很多：怎么寻找依赖并输出文件、如果开发过程使用了浏览器不兼容语法如何编译、如何管理文件名（文件名称涉及缓存）、如何提取公共代码减小文件体积、css/图片/字体资源如何管理，等等，webpack 都可以解决。

## webpack 背后的逻辑

通过上面两点可以这么说，webpack 是在前端工程化浪潮下一个必然产物，没有 webpack 也会产生其他工具。并且随着 es6 的发展，之前非官方支持的标准，比如 CMD、AMD 可以已经走到的生命的尽头，后面完全可以使用 es6 的模块语法几 commonJS（node）结合 webpack 使用。

## 第三方支持

像是 angular、react、vue 本身就集成了 webpack，所以从侧面逼着自己学。
